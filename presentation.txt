## â“ Anticipated Questions & Answers

### Q: Why did you choose Random Forest over other models? 

> "Random Forest was chosen because it: 
> - Handles non-linear relationships well
> - Is robust to overfitting with proper hyperparameters
> - Provides interpretable feature importance
> - Requires minimal hyperparameter tuning for reasonable performance
>
> For production, I would also test gradient boosting methods (XGBoost, LightGBM) and potentially LSTM networks for longer-term forecasting."

### Q: How would you improve the prediction model?

> "Several improvements are possible:
> - **Additional features**:  Weather data (temperature, wind), calendar features (weekday/weekend, holidays), lagged prices
> - **Ensemble methods**: Combine multiple models for more robust predictions
> - **Hyperparameter tuning**: Use cross-validation to optimise model parameters
> - **Real-time updates**: Implement streaming data for live predictions"

### Q: How would this integrate with a trading system?

> "This dashboard could be extended to:
> - Generate buy/sell signals based on forecast spreads
> - Calculate optimal dispatch schedules for flexible assets
> - Provide risk metrics for position sizing
> - Integrate with order management systems via API"

### Q: What are the limitations of this analysis?

> "Key limitations include:
> - **7-day data window**:  Elexon API limit restricts historical analysis
> - **Simple features**: More sophisticated feature engineering would improve predictions
> - **No weather data**: Temperature is a key driver of demand
> - **Point forecasts only**: Probabilistic forecasts would be more useful for risk management"

### Q: How did you validate the model?

> "I used several validation approaches:
> - **Persistence baseline**:  Comparing against a naÃ¯ve 'tomorrow = today' forecast
> - **Time series cross-validation**:  Using `TimeSeriesSplit` to avoid look-ahead bias
> - **Multiple metrics**: MAE, RMSE, and bias to capture different error characteristics"
